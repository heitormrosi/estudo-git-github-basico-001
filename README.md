# Estudo de Git e GitHub

## Estudo e implementação de exemplos e exercícios dos livros "Controlando Versões com Git e GitHub", de Alexandre Aquiles e Rodrigo Ferreira, e "Pro Git", de Scott Chacon e Ben Straub

## Descrição dos livros

> As fontes exploradas detalham a origem histórica, o funcionamento técnico e a aplicação prática do Git como um sistema de controle de versão distribuído. Criado por Linus Torvalds para superar as limitações de ferramentas centralizadas, o Git se diferencia por registrar instantâneos de dados em vez de apenas variações de arquivos. O conteúdo abrange desde procedimentos de instalação e configuração em diversos sistemas operacionais até comandos fundamentais para colaboração em repositórios, como clonagem, criação de forks e automação via hooks. Além disso, os textos explicam a arquitetura interna baseada em objetos e o uso de interfaces externas, como o GitHub, para gerenciar projetos de código aberto. Por fim, são apresentadas ferramentas avançadas de personalização e análise, incluindo bibliotecas de programação e recursos de busca em bases de código complexas.
